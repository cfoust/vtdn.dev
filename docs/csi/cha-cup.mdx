**CHA** (Cursor Character Absolute) and **CUP** (Cursor Position) are CSI sequences for absolute cursor positioning. CHA sets the column, while CUP sets both the row and column.

## Syntax

```
CSI Pn G          CHA — Cursor Character Absolute (set column)
CSI Pn1 ; Pn2 H   CUP — Cursor Position (set row and column)
```

<FormalSyntax grammar={`

### Parameters

**CHA:**

| Parameter | Description | Default |
|-----------|-------------|---------|
| `Pn` | Column number | 1 |

**CUP:**

| Parameter | Description | Default |
|-----------|-------------|---------|
| `Pn1` | Row number | 1 |
| `Pn2` | Column number | 1 |

## Description

Per ECMA-48 §8.3.9 (CHA) and §8.3.21 (CUP):

- **CHA** (`CSI Pn G`) — Moves the cursor to column `Pn` of the current line. The row does not change.
- **CUP** (`CSI Pn1;Pn2 H`) — Moves the cursor to row `Pn1`, column `Pn2`. `CSI H` with no parameters moves to the home position (row 1, column 1).

Positions are 1-based. Values exceeding the screen dimensions are clamped to the maximum row or column.

### Interaction with DECOM

When Origin Mode (DECOM) is set, CUP coordinates are relative to the top margin of the scrolling region rather than to the top of the screen, and the cursor is constrained within the scrolling region. When DECOM is reset (the default), coordinates are absolute from the top-left of the screen.

## Examples

```bash
printf '\e[1;1H'     # Move to top-left corner (home)
printf '\e[H'        # Same — move to home (defaults)
printf '\e[10;20H'   # Move to row 10, column 20
printf '\e[5G'       # Move to column 5 on current line
```
