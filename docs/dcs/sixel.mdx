**Sixel graphics** is a DCS-based bitmap graphics protocol originally introduced on the DEC VT240 and VT340 terminals. It encodes images as rows of six vertical pixels (a "sixel"), transmitted inline within the terminal data stream.

## Syntax

```
DCS P1 ; P2 ; P3 q sixel-data ST    Sixel graphics sequence
```

<FormalSyntax grammar={`

P2 = digit, { digit } ;             (* background select: 0 or 2 = device default, 1 = no change *)
P3 = digit, { digit } ;             (* horizontal grid size in pixels, default: 0 *)

Pad = digit, { digit } ;            (* pixel aspect denominator *)

LF = "-" ;                           (* line feed — advance down one sixel row *)

      | "5" | "6" | "7" | "8" | "9" ;

### Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `P1` | Pixel aspect ratio selector (see table below) | `0` |
| `P2` | Background color select: `0` or `2` = use terminal default, `1` = do not erase background | `0` |
| `P3` | Horizontal grid size in pixels (ignored by most modern terminals) | `0` |

#### P1 aspect ratio values

| Value | Aspect ratio (vertical:horizontal) |
|-------|-----------------------------------|
| `0`, `1` | 2:1 |
| `2` | 5:1 |
| `3`, `4` | 3:1 |
| `5`, `6` | 2:1 |
| `7`, `8`, `9` | 1:1 |

Most modern terminal emulators ignore `P1` and render with 1:1 pixel aspect ratio.

## Description

Sixel graphics encode images as a series of vertical six-pixel columns. Each sixel character (in the range `?` to `~`, ASCII 0x3F to 0x7E) represents a column of six pixels, where bit 0 is the top pixel and bit 5 is the bottom pixel. The character's value minus 0x3F gives the six-bit pattern.

For example, the character `~` (0x7E) has value 63 (0x3F subtracted), which is `111111` in binary -- all six pixels on. The character `?` (0x3F) has value 0 -- all six pixels off.

### Raster attributes

The optional raster attributes string `"Pan;Pad;Ph;Pv` specifies the pixel aspect ratio (`Pan:Pad`) and image dimensions (`Ph` x `Pv` pixels). Modern terminals use this to allocate the image canvas before rendering begins.

### Color introduction

The `#` character introduces a color command:

- **Color selection**: `#Pc` selects color register `Pc` for subsequent sixel data.
- **Color definition**: `#Pc;Pu;Px;Py;Pz` defines a color register. `Pu=1` specifies HLS (Hue/Lightness/Saturation) coordinates; `Pu=2` specifies RGB percentages (0-100 for each component).

### Repeat introducer

The `!` character introduces a repeat command: `!Pn C` repeats sixel character `C` a total of `Pn` times. This provides simple run-length compression, which significantly reduces data size for images with horizontal runs of identical pixel columns.

### Carriage return and line feed

- `$` (Graphics Carriage Return): Moves the drawing position back to the left edge of the current sixel row, allowing overprinting with additional colors.
- `-` (Graphics Line Feed): Advances the drawing position down by one sixel row (six pixels) and returns to the left edge.

### Background handling

The `P2` parameter controls whether the terminal erases the background behind the image. When `P2=1`, pixels with value 0 in the sixel data are left unchanged (transparent). When `P2=0` or `P2=2`, background pixels are filled with the terminal's default background color.

## Examples

```bash
# Draw a simple 1x6 red pixel column
# Define color 1 as red (RGB: 100,0,0), then draw a full sixel column
printf '\ePq#1;2;100;0;0#1~\e\\'

# A 10-pixel-wide horizontal red bar, 6 pixels tall
printf '\ePq#1;2;100;0;0#1!10~\e\\'

# Two-color image with raster attributes (10x12 pixels)
# Red top half, blue bottom half
printf '\ePq"1;1;10;12#1;2;100;0;0#1!10~-#2;2;0;0;100#2!10~\e\\'
```
