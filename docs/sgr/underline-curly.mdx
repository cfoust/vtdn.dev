**Curly underline** (SGR 4:3) is an extended underline style that renders a wavy or curly line below the text.

## Syntax

```
CSI 4:3 m    SGR 4:3 — Curly/wavy underline
CSI 24 m     SGR 24 — Not underlined
```

<FormalSyntax grammar={`
SGR-curly-underline = 0x1b, "[", "4:3", "m" ;
SGR-not-underline   = 0x1b, "[", "24", "m" ;
`} />

## Description

SGR 4:3 selects a curly (wavy) underline style using the sub-parameter syntax from ITU-T T.416. This extends the basic underline (SGR 4) with additional style variants.

The sub-parameter values for SGR 4 are:

| Sub-parameter | Style |
|---------------|-------|
| `4:0` | No underline |
| `4:1` | Single straight underline (same as SGR 4) |
| `4:2` | Double underline (same as SGR 21) |
| `4:3` | Curly/wavy underline |
| `4:4` | Dotted underline |
| `4:5` | Dashed underline |

Curly underlines are reset by SGR 24 or SGR 0.

Curly underlines are commonly used by terminal applications to indicate spelling errors, diagnostics, or other inline annotations — similar to the "squiggly underline" in code editors. The underline color can be set independently using SGR 58.

## Examples

```bash
printf '\e[4:3mCurly underlined text\e[24m Normal text\n'
printf '\e[4:3;58;5;9mRed curly underline\e[0m\n'
```
